# Dataiku Runtime Image Build

This repository builds the internal Dataiku DSS runtime image used by the
Kubernetes/Argo CD deployment.

The build intentionally does not depend on a production-ready Dataiku image
from the vendor. It consumes the official Dataiku DSS kit archive and produces
an internal OCI image:

```text
registry.internal/dataiku/dss-runtime:<DSS_VERSION>-<BUILD_REV>
```

## Inputs

- `dataiku-dss-<DSS_VERSION>.tar.gz`: official Dataiku DSS kit.
- Optional files under `assets/`: internal CA certificates, JDBC/ODBC drivers,
  plugins, and Spark/Hadoop archives.
- No license, password, token, or private key is baked into the image.

## Build

```bash
export DSS_VERSION=14.x.y
export BUILD_REV=1
export REGISTRY=registry.internal/dataiku

./scripts/build-runtime.sh
./scripts/scan-image.sh "${REGISTRY}/dss-runtime:${DSS_VERSION}-${BUILD_REV}"
./scripts/push-image.sh "${REGISTRY}/dss-runtime:${DSS_VERSION}-${BUILD_REV}"
```

If the DSS kit is not already present in the build context, fetch it first:

```bash
./scripts/fetch-kit.sh
```

Set `DATAIKU_DSS_KIT_URL` when your organization mirrors Dataiku kits in an
internal artifact repository.

The resulting tag must be promoted into the GitOps repository by updating:

```text
dataiku-gitops/envs/onprem/prod/values.yaml
```

Argo CD only deploys approved image tags. It does not build images.

## Runtime Behavior

The image contains the extracted DSS install directory under:

```text
/opt/dataiku/dataiku-dss-<DSS_VERSION>
```

At container startup, `entrypoint.sh` initializes or upgrades the mounted
`DATA_DIR`, then runs DSS in foreground mode.

Supported `DATAIKU_NODE_TYPE` values:

- `design`
- `automation`
- `deployer`
- `govern`

For `govern`, PostgreSQL connection settings are injected from Kubernetes
Secrets generated by External Secrets Operator.

## Important Operational Notes

- `DATA_DIR` must be mounted on POSIX block storage such as ext4 or XFS.
- Do not use NFS, GlusterFS, EFS, or other non-POSIX shared filesystems for
  `DATA_DIR`.
- After every DSS upgrade, rebuild this runtime image and all Dataiku execution
  images (`container-exec`, `spark`, `cde`, `api-deployer`).
